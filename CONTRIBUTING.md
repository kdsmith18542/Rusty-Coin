# Contributing to Rusty-Coin

Thank you for your interest in contributing to Rusty-Coin! We welcome contributions from the community to help make this project better.

## How to Contribute

### Reporting Bugs

- Search the existing issues to see if the bug has already been reported.
- If not, create a new issue with a clear title and description.
- Include as much information as possible, such as logs, stack traces, and steps to reproduce the bug.

### Suggesting Enhancements

- Search the existing issues to see if the enhancement has already been suggested.
- If not, create a new issue describing the enhancement and why it would be useful.

### Pull Requests

1. Fork the repository.
2. Create a new branch for your changes (`git checkout -b feature/my-feature`).
3. Make your changes and ensure that all tests pass (`cargo test`).
4. Commit your changes with clear and descriptive commit messages.
5. Push your changes to your fork.
6. Submit a pull request to the `master` branch.

## Coding Standards

- Follow standard Rust formatting conventions (use `cargo fmt`).
- Ensure your code is well-documented with docstrings.
- Write tests for new features and bug fixes.
- Keep PRs focused on a single change.

## Development Environment

We recommend using `rust-analyzer` with your preferred editor (VS Code, Vim, Emacs, etc.).

## Community

Join our community discussions on [GitHub Discussions](https://github.com/kdsmith18542/Rusty-Coin/discussions) or reach out to the maintainers.
